## Understanding User Privileges in Linux

One of the most fundamental concepts in Linux is user privileges. Linux is a multi-user system where different users have different levels of access to system resources.

### The Three Types of Users

<div className="bg-gradient-to-r from-blue-50 to-indigo-50 dark:from-blue-900 dark:to-indigo-900 p-6 rounded-lg my-6">

#### 1. Root User (Superuser)
- Username: `root`
- UID (User ID): 0
- Has **unlimited power** over the entire system
- Can modify any file, kill any process, change any setting
- **Dangerous**: A mistake as root can break the entire system

#### 2. Regular Users
- UID: 1000 and above (on modern systems)
- Limited permissions - can only modify their own files
- Cannot install system-wide software or change system settings
- Safe for daily operations

#### 3. System Users
- UID: 1-999 (typically)
- Used by services and daemons (e.g., `apache`, `mysql`, `sshd`)
- Cannot login interactively
- Run system services with minimal privileges

</div>

## Understanding Important Groups: wheel and sudo

<div className="bg-yellow-50 dark:bg-yellow-900/20 p-6 rounded-lg my-6">

## Privilege Escalation: `su` vs `sudo`

There are two primary ways to gain administrative privileges in Linux: `su` and `sudo`. Understanding when and how to use each is crucial for system administration.

### Method 1: `su -` (Switch User)

The `su` command allows you to **switch user** accounts.

**Basic Usage - Switching to Root:**

```bash
su -
```

**What happens:**

```bash
Password: [enter root password]
[root@centos ~]#
```

**Command Breakdown:**
- `su`: Switch user command
- `-`: Login shell option (important!)
- Without username specified, defaults to `root`
- Prompts for root password
- Notice the prompt changes from `$` to `#` (indicating root user)

**Why the hyphen `-` matters:**

<div className="grid grid-cols-1 md:grid-cols-2 gap-6 my-8">

<div className="bg-white dark:bg-gray-800 p-6 rounded-lg border border-gray-200 dark:border-gray-700">

#### `su` (without hyphen)
```bash
su
```
- Does NOT load root's environment
- Stays in current directory
- Uses current user's PATH
- Keeps current user's shell settings
- **Not recommended** for admin tasks

</div>

<div className="bg-white dark:bg-gray-800 p-6 rounded-lg border border-gray-200 dark:border-gray-700">

#### `su -` (with hyphen)
```bash
su -
```
- Loads complete root environment
- Changes to root's home directory (`/root`)
- Uses root's PATH
- Loads root's shell configuration
- **Recommended** for admin work

</div>

</div>

**Verifying You're Root:**

After switching to root with `su -`, verify your identity:

```bash
id
```

**Output:**
```
uid=0(root) gid=0(root) groups=0(root) context=unconfined_u:unconfined_r:unconfined_t:s0-s0:c0.c1023
```

**Key Indicators You're Root:**
- `uid=0` - Root always has UID 0
- `gid=0` - Root's primary group is also 0
- `groups=0(root)` - Member of root group
- Shell prompt shows `#` instead of `$`
- Prompt shows `[root@hostname ~]#`

**Exiting Root Shell:**

When done with administrative tasks, return to your regular user:

```bash
exit
```

**Output:**
```
logout
[centos9@centos ~]$
```

Notice the prompt changes back to showing your regular username and the `$` symbol.

<Callout type="warning">
**Security Best Practice**: Never stay logged in as root longer than necessary! Always `exit` back to your regular user after completing administrative tasks.
</Callout>

### Method 2: `sudo` (Super User Do)

The `sudo` command allows you to execute **individual commands** with root privileges without switching to the root account.

**Basic Usage:**

```bash
sudo ls /root
```

**What happens:**

```bash
[sudo] password for centos9: [enter YOUR password, not root's]
anaconda-ks.cfg
```

**Command Breakdown:**
- `sudo`: Execute following command as root
- `ls /root`: The command to execute (list files in /root directory)
- Prompts for **your password** (not root password!)
- Executes just this one command as root
- Returns you to regular user immediately

**Why This Works:**

Regular users cannot access `/root`:

```bash
ls /root
```

**Output:**
```
ls: cannot open directory '/root': Permission denied
```

But with `sudo`:

```bash
sudo ls /root
```

**Output:**
```
anaconda-ks.cfg
```

Success! The command executed with root privileges.

## Understanding Command Syntax and Case Sensitivity

Linux commands follow specific syntax rules that are different from Windows. Understanding these rules is essential for effective command-line usage.

### Linux is Case Sensitive

Unlike Windows, Linux treats uppercase and lowercase as completely different characters.

**Demonstration:**

```bash
ls
```

**Output:**
```
Coursera  Desktop  Documents  Downloads  Music  Pictures  Public  Templates  Videos
```

This works fine. But watch what happens with uppercase:

```bash
LS
```

**Output:**
```
bash: LS: command not found...
Similar command is: 'ls'
```

**Try with mixed case:**

```bash
Ls
```

**Output:**
```
bash: Ls: command not found...
Similar command is: 'ls'
```

**Why This Matters:**
- Commands must be typed in correct case: `ls`, not `LS` or `Ls`
- Filenames are case-sensitive: `file.txt` ‚â† `File.txt` ‚â† `FILE.TXT`
- Directory names are case-sensitive: `/home/user` ‚â† `/Home/User`

<div className="bg-gradient-to-r from-red-50 to-orange-50 dark:from-red-900 dark:to-orange-900 p-6 rounded-lg my-6">

**Common Beginner Mistakes:**
- Typing `CD` instead of `cd`
- Using `PWD` instead of `pwd`
- Typing `SUDO` instead of `sudo`
- Creating files with different cases and getting confused

</div>

### Command-Line Option Syntax

Linux commands support two types of options: **short options** and **long options**.

#### Short Options (Single Hyphen)

Short options use a single hyphen `-` followed by a single letter:

```bash
ls -a
ls -l
ls -h
```

**You can combine multiple short options:**

```bash
ls -al
# Same as: ls -a -l
```

```bash
ls -lh
# Same as: ls -l -h
```

#### Long Options (Double Hyphen)

Long options use double hyphens `--` followed by a descriptive word:

```bash
ls --all
ls --help
mkdir --parents
```

**Visual Example - Common Mistake:**

```bash
mkdir -help
```

**Output:**
```
mkdir: invalid option -- 'h'
Try 'mkdir --help' for more information.
```

**Why it failed:** Single hyphen `-help` is interpreted as short options: `-h`, `-e`, `-l`, `-p`. Since `mkdir` doesn't have a `-h` option, it fails.

**Correct usage:**

```bash
mkdir --help
```

**Output (truncated):**
```
Usage: mkdir [OPTION]... DIRECTORY...
Create the DIRECTORY(ies), if they do not already exist.

Mandatory arguments to long options are mandatory for short options too.
  -m, --mode=MODE   set file mode (as in chmod), not a=rwx - umask
  -p, --parents     no error if existing, make parent directories as needed
  -v, --verbose     print a message for each created directory
  ...
```

**Key Takeaway:**
- Short options: `-a`, `-l`, `-h` (single hyphen, single letter)
- Long options: `--all`, `--help`, `--parents` (double hyphen, word)
- Most commands support both formats for common options

## Essential Navigation and File Commands

Now that we understand user privileges and command syntax, let's explore essential commands for navigating the filesystem and working with files.

### The `pwd` Command: Print Working Directory

Always know where you are in the filesystem:

```bash
pwd
```

**Output:**
```
/home/centos9
```

**What This Means:**
- You are currently in the `/home/centos9` directory
- This is your home directory (where you land after logging in)
- The path is **absolute** - starts from root `/`

**Command Breakdown:**
- `pwd`: Print Working Directory
- No arguments needed
- Always shows your current location in the filesystem

### The `cd` Command: Change Directory

Navigate through the filesystem:

**Example 1: Navigate to /var/log**

```bash
cd /var/log
pwd
```

**Output:**
```
/var/log
```

**What Happened:**
- `cd /var/log`: Changed to the `/var/log` directory
- `pwd`: Confirmed we're now in `/var/log`
- `/var/log` is where system log files are stored

**Example 2: Return Home**

```bash
cd
pwd
```

**Output:**
```
/home/centos9
```

**What Happened:**
- `cd` with no arguments takes you to your home directory
- Confirmed with `pwd` - we're back in `/home/centos9`

**Alternative Ways to Go Home:**

```bash
cd ~      # Tilde (~) represents your home directory
cd $HOME  # Environment variable for home directory
cd        # No arguments - default behavior
```

All three methods take you to your home directory (`/home/centos9` for user centos9).

### The `touch` Command: Create Empty Files

The `touch` command creates empty files or updates timestamps:

**Creating a New File:**

```bash
touch hello
ls -l hello
```

**Output:**
```
-rw-r--r--. 1 centos9 centos9 0 Oct 31 15:24 hello
```

**What Happened:**
- `touch hello` created an empty file named `hello`
- `ls -l hello` shows file details
- Size is 0 bytes (empty file)
- File created at 15:24 on Oct 31

**Updating Timestamp:**

Wait a minute, then run `touch` again:

```bash
touch hello
ls -l hello
```

**Output:**
```
-rw-r--r--. 1 centos9 centos9 0 Oct 31 15:25 hello
```

**Notice:** The timestamp updated from 15:24 to 15:25! The file still exists and is still empty, but the "last modified" time has changed.

**Why `touch` is Useful:**
- Create placeholder files quickly
- Test file creation permissions
- Update file modification times (important for backup scripts)
- Create multiple files at once: `touch file1 file2 file3`

## The `passwd` Command: Change Your Password

Changing your password is a fundamental security practice:

```bash
passwd
```

**Interactive Process:**
```
Changing password for user centos9.
Current password: [type your current password]
New password: [type your new password]
Retype new password: [type your new password again]
passwd: all authentication tokens updated successfully.
```

**Password Requirements:**
- Minimum length (usually 8+ characters)
- Mix of uppercase, lowercase, numbers, symbols
- Not based on dictionary words
- Not similar to username
- Not a recently used password

**Common Error:**

```bash
passwd
```

```
Changing password for user centos9.
Current password:
New password:
BAD PASSWORD: The password is the same as the old one
passwd: Authentication token manipulation error
```

**Why It Failed:** Trying to use the same password. Linux enforces password change policies!

**Viewing `passwd` Help:**

```bash
passwd --help
```

**Output:**
```
Usage: passwd [OPTION...] <accountName>
  -k, --keep-tokens       keep non-expired authentication tokens
  -d, --delete            delete the password for the named account (root only)
  -l, --lock              lock the password for the named account (root only)
  -u, --unlock            unlock the password for the named account (root only)
  -e, --expire            expire the password for the named account (root only)
  -f, --force             force operation
  -x, --maximum=DAYS      maximum password lifetime (root only)
  -n, --minimum=DAYS      minimum password lifetime (root only)
  -w, --warning=DAYS      number of days warning users receives before password expiration (root only)
  -i, --inactive=DAYS     number of days after password expiration when an account becomes disabled (root only)
  -S, --status            report password status on the named account (root only)
      --stdin             read new tokens from stdin (root only)
```

**Important Options:**
- Most options require root privileges (`root only`)
- Regular users can only change their own password
- Root can change any user's password: `sudo passwd username`
- Root can lock/unlock accounts, set password policies

## Best Practices and Common Mistakes

### Best Practices

<div className="bg-gradient-to-br from-green-50 to-blue-50 dark:from-green-900 dark:to-blue-900 p-6 rounded-lg my-8">

‚úÖ **DO:**
1. Use `sudo` for individual administrative commands
2. Use `--help` before using unfamiliar commands
3. Always use `pwd` to confirm your location before file operations
4. Use `ls -la` to see complete directory contents including hidden files
5. Return to regular user with `exit` immediately after administrative tasks
6. Pay attention to shell prompt - `$` (user) vs `#` (root)
7. Use tab completion to avoid typos
8. Check command case sensitivity

</div>

### Common Mistakes to Avoid

<div className="bg-gradient-to-br from-red-50 to-orange-50 dark:from-red-900 dark:to-orange-900 p-6 rounded-lg my-8">

‚ùå **DON'T:**
1. Stay logged in as root longer than necessary
2. Forget which user you're operating as
3. Use uppercase letters in commands (e.g., LS instead of ls)
4. Mix up short and long option syntax (-help vs --help)
5. Ignore "Permission denied" errors - use sudo when appropriate
6. Assume commands are the same across all operating systems
7. Skip reading `--help` output before using complex commands
8. Work in /root directory - use your home directory instead

</div>
